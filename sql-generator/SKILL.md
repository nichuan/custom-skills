# SQL生成助手

## 使用流程

当用户提出以下需求时使用本Skill:
- 生成或修改SQL查询语句
- 查询表结构或表之间的关联关系
- 基于已有SQL模板调整查询逻辑
- 需要分析特定业务场景的数据查询

## 执行步骤

1. **识别涉及的表**: 从用户需求中提取表名,对照 `table_meta.md` 确认表是否存在
2. **按需加载详细结构**: 仅读取用户需求中涉及的 `table_detail/[表名].md` 文件
3. **补充关联关系**: 参考 `relations.md` 确认表之间的关联键
4. **应用SQL模板**: 从 `sql_templates.md` 中匹配适用的通用模板
5. **生成/调整SQL**: 结合表结构、关联关系和模板,生成符合语法的SQL
6. **标注信息**: 在SQL注释中标注使用的表、关联关系和核心字段

## 核心规则

### SQL生成规则
- 必须先确认表关联键的正确性(参考 `relations.md`)
- 使用明确的表别名避免字段歧义
- 时间范围过滤优先使用 `sql_templates.md` 中的模板
- 聚合统计时明确 GROUP BY 字段和聚合函数
- 生成的SQL必须包含关键注释说明

### 上下文加载规则
- 严格按需加载,只读取 `table_detail/[表名].md` 中涉及的表
- 不加载全量表结构,保持上下文精简
- 如果某个表的详细结构不存在,仅使用 `table_meta.md` 中的核心字段

### 准确性保障
- 生成SQL前必须验证:
  - 表名拼写是否正确(对照 `table_meta.md`)
  - 关联键是否匹配(对照 `relations.md`)
  - 字段是否存在于对应表(对照 `table_detail/[表名].md`)
- 对于复杂的查询逻辑,建议分步生成并验证

## 参考文件指引

### 核心元数据
- **表极简元数据**: `references/table_meta.md`
  - 所有表的名称、核心字段、主键、关联键
  - 用途: 快速匹配表名和确认基本信息

### 表详细信息
- **单表详细结构**: `references/table_detail/[表名].md`
  - 替换 `[表名]` 为具体的表名
  - 用途: 获取完整字段列表、字段类型、字段说明
  - 示例: `references/table_detail/user.md`

### 关联关系
- **表关联关系**: `references/relations.md`
  - 高频表之间的关联键和关联类型
  - 用途: 确保表关联的正确性

### SQL模板
- **通用SQL模板**: `references/sql_templates.md`
  - 时间范围过滤、表关联、聚合统计等通用模板
  - 用途: 快速生成符合规范的SQL片段

### SQL示例
- **完整SQL示例**: `assets/sql_template_examples/[场景名].sql`
  - 按业务场景分类的完整SQL文件
  - 用途: 参考类似场景的SQL写法

## 输出格式

生成的SQL应包含以下信息:
```sql
-- 业务场景: [描述用户需求]
-- 涉及表: [表1, 表2, ...]
-- 关联关系: [表1.key = 表2.key, ...]
-- 核心字段: [主要查询和统计字段]

SELECT ...
FROM ...
WHERE ...
GROUP BY ...
```

## 常见场景

### 场景1: 生成带时间范围过滤的查询
- 从 `sql_templates.md` 匹配时间过滤模板
- 结合 `table_detail/[表名].md` 确认时间字段名称
- 标注时间范围和过滤条件

### 场景2: 多表关联查询
- 从 `relations.md` 确认所有关联关系
- 为每个表设置清晰的别名
- 在注释中列出所有关联键

### 场景3: 聚合统计查询
- 从 `sql_templates.md` 匹配聚合模板
- 确认 GROUP BY 字段的准确性
- 标注聚合逻辑和统计指标

## 错误处理

如果遇到以下情况,应明确告知用户:
- 请求的表在 `table_meta.md` 中不存在
- 表关联关系在 `relations.md` 中未定义且无法推断
- 某个表缺少 `table_detail/[表名].md` 详细结构文件
- 用户需求过于模糊,需要进一步澄清

## 扩展指南

### 新增表
1. 在 `table_meta.md` 中添加表的极简元数据
2. 创建 `references/table_detail/[表名].md` 文件
3. 如有高频关联,在 `relations.md` 中补充关联关系

### 新增SQL模板
1. 将通用SQL模板添加到 `sql_templates.md`
2. 按模板类型分类(如: 时间过滤、关联查询、聚合统计)
3. 提供清晰的占位符说明

### 新增SQL示例
1. 在 `assets/sql_template_examples/` 中创建 `[场景名].sql`
2. 使用清晰的命名描述业务场景
3. 在文件头部添加注释说明业务逻辑
